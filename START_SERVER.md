# تشغيل السيرفر (Backend) للداشبورد

## سبب ظهور 404 للـ vendors, paints, colors, إلخ

الطلبات ترجع 404 لأن **البروسيس اللي شغال على المنفذ 5000 مش من المشروع ده** (نسخة قديمة أو برنامج تاني).

## الخطوات الصحيحة

### 1. إيقاف أي سيرفر قديم على المنفذ 5000

في **PowerShell** (كمسؤول لو لزم):

```powershell
# عرض العملية اللي تستخدم المنفذ 5000
netstat -ano | findstr :5000

# ستظهر سطر مثل: TCP    0.0.0.0:5000    0.0.0.0:0    LISTENING    12345
# الرقم الأخير (12345) هو الـ PID

# إيقاف العملية (استبدل 12345 بالـ PID اللي ظهر عندك)
taskkill /PID 12345 /F
```

أو: اقفل أي تيرمنال/نافذة فيها سيرفر Node قديم واضغط **Ctrl+C** فيها.

### 2. تشغيل السيرفر من مجلد المشروع

```powershell
cd E:\PaintApp\PaintApp_Backend
npm run dev
```

### 3. التحقق من أن السيرفر الصحيح يعمل

- في التيرمنال لازم تظهر:
  - `Server running on port 5000`
  - `Project root: E:\PaintApp\PaintApp_Backend` (أو مسار مشابه)
  - سطر فيه: `Dashboard API: GET /users, /paints, /vendors...`

- في المتصفح افتح:
  ```
  http://localhost:5000/api/ping
  ```
  لو ظهر شيء مثل: `{"ok":true,"routes":"users,paints,vendors,..."}` يبقى السيرفر المحدّث شغال.

### 4. تحديث صفحة الداشبورد

حدّث الصفحة (F5) أو أعد فتح الداشبورد؛ المفروض تظهر البيانات (مستخدمين، بائعين، منتجات، إلخ).
